# Installation Guide for KeyHunter Framework

This guide provides step-by-step instructions to set up and install the KeyHunter Framework on your system. It is recommended to use a Linux-based environment, such as Kali Linux, for optimal compatibility.

## Prerequisites

Before you begin, ensure you have the following installed:

*   **Python 3.8+**: KeyHunter Framework is developed with Python 3.8 and later versions.
*   **Git**: For cloning the repository.

## Step 1: Clone the Repository

First, clone the KeyHunter Framework repository from GitHub to your local machine:

```bash
git clone https://github.com/Panda1847/keyhunter-framework.git
cd keyhunter-framework
```

## Step 2: Run the Installer Script

The repository includes an `install.sh` script that automates the setup process, including creating a Python virtual environment and installing all necessary dependencies.

```bash
chmod +x hybrid_scraper/install.sh
./hybrid_scraper/install.sh
```

This script will perform the following actions:

1.  Create a Python virtual environment named `venv`.
2.  Activate the virtual environment.
3.  Install all required Python packages listed in `requirements.txt` (which will be generated by the script).

## Step 3: Verify Installation

After the installation script completes, you can verify the setup by activating the virtual environment and checking the installed packages:

```bash
source venv/bin/activate
pip list
```

You should see `curl_cffi`, `nodriver`, `loguru`, and `aiohttp` among the installed packages.

## Manual Installation (Optional)

If you prefer to set up the environment manually, follow these steps:

1.  **Create a Virtual Environment**:
    ```bash
    python3 -m venv venv
    source venv/bin/activate
    ```

2.  **Install Dependencies**:
    Create a `requirements.txt` file with the following content:
    ```
    curl_cffi
nodriver
loguru
aiohttp
    ```
    Then install them using pip:
    ```bash
    pip install -r requirements.txt
    ```

## Troubleshooting

*   **Permission Denied**: If you encounter a "Permission denied" error when running `install.sh`, ensure you have executed `chmod +x hybrid_scraper/install.sh`.
*   **Python Version**: If you have multiple Python versions, ensure `python3` points to Python 3.8+.
*   **Network Issues**: Proxy fetching might fail if there are network connectivity issues. Ensure your internet connection is stable.

For further assistance, please refer to the [KeyHunter Framework GitHub Issues](https://github.com/Panda1847/keyhunter-framework/issues) page.
