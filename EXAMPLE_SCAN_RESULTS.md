# Example Scan Results

This section provides examples of the output generated by the KeyHunter Framework during its operation. These examples illustrate the key discovery process and the type of information that can be obtained.

## Command Line Interface (CLI) Output

The primary interface for KeyHunter Framework is the command line. Below is an example of the CLI output during an active scan, showcasing the discovery of various API keys.

![KeyHunter CLI Output](/home/ubuntu/keyhunter-framework/screenshot_cli.png)

In this example, the framework successfully identified API keys for services such as Tavily AI and Shodan, along with a generic API key. The `SUCCESS` messages indicate validated keys.

## Dashboard Visualization (Conceptual)

For more advanced deployments or integrations, KeyHunter Framework can feed its data into a dashboard for real-time visualization and analysis. This conceptual dashboard illustrates how discovered keys, proxy usage, and geographic distribution could be presented.

![KeyHunter Dashboard Visualization](/home/ubuntu/keyhunter-framework/screenshot_dashboard.png)

This dashboard provides an overview of:

*   **Global Proxy Map**: Visualizing the geographic distribution of proxies used.
*   **Real-Time Key Discovery**: A live feed of newly found API keys.
*   **Keys Found by Service**: Charts (bar and pie) illustrating the distribution of discovered keys across different services.

These visualizations help in understanding the scope of the scan and the types of credentials being exposed.
